Governance: AI Design Rules

When modifying Beaver Board UI:

  • Use existing design tokens. Do not introduce raw hex colors (`#FF0000`, `#333`) in components.
  • Follow Agent Status strictly. Running is Blue, Blocked is Red, Assigned/Waiting is Amber, Completed is Green.
  • Kanban is sacred. Do not break the grid. Cards must remain compact and developer-readable.
  • No KittyClaw branding. Do not use KittyClaw visual identity, old logos, or references in the UI.
  • Evidence is required. Any task completion state must show how evidence (tests/diffs) is presented.
  • Keep it a dev tool. Avoid generic SaaS styles, big bubbly buttons, or childish mascot overload. It must look like a tool for GitHub/terminal users.

PR Review Checklist (Design Gate)

  • Uses `var(--color-bg-panel)` instead of `#111113`?
  • Spacing uses `var(--space-X)` variables?
  • Does the new feature have Empty, Loading, and Error states?
  • Are agent statuses using the semantic chips?
  • Is the contrast ratio WCAG AA compliant on dark backgrounds?
  • Does drag-and-drop have a keyboard alternative?
  • Is the focus outline (`:focus-visible`) intact?
